## 🚀 DEPLOY TO RAILWAY IN 5 STEPS

### Step 1: Push to GitHub
```bash
# If not already pushed
git push origin master

# Or if remote doesn't exist:
git remote set-url origin https://github.com/YOUR_USERNAME/ai-prompt-library.git
git push -u origin master
```

### Step 2: Go to Railway.app
1. Visit https://railway.app
2. Click "Login" → Use GitHub account
3. Click "New Project"
4. Select "Deploy from GitHub repo"
5. Authorize & select `ai-prompt-library`

### Step 3: Wait for Auto-Detection
Railway will automatically detect:
- ✅ PostgreSQL database
- ✅ Redis cache
- ✅ Backend (Django/Python)
- ✅ Frontend (Angular/Node)

### Step 4: Set Environment Variables
Click each service and add these variables:

**Backend Service:**
```
DEBUG=False
SECRET_KEY=<GENERATE-KEY-BELOW>
ALLOWED_HOSTS=<your-domain>.railway.app,yourdomain.com
DB_NAME=prompts
DB_USER=postgres
DB_PASSWORD=<auto-generated-by-railway>
```

**Generate SECRET_KEY (copy & paste output):**
```bash
python -c 'from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())'
```

### Step 5: Click "Deploy"
Railway will:
1. Build Docker images
2. Start all services
3. Run migrations automatically
4. Deploy live

✅ **Your app is now live!**

## 🔧 SETUP CUSTOM DOMAIN (Optional)

1. **In Railway Dashboard:**
   - Select Frontend service
   - Click "Settings" → "Domain"
   - Add your domain (e.g., `prompt-library.com`)

2. **In Your Domain Provider (GoDaddy, Namecheap, etc.):**
   - Update DNS with CNAME record:
     ```
     CNAME  www  → your-railway-domain.railway.app
     CNAME  @    → your-railway-domain.railway.app
     ```

3. **Wait 5-30 minutes** for DNS propagation

SSL/TLS is automatic! 🔒

## 🛠️ USEFUL COMMANDS

### View Logs
```bash
railway logs              # All services
railway logs -s backend   # Just backend
```

### Run Migrations
```bash
railway exec backend python manage.py migrate
```

### Create Admin User
```bash
railway exec backend python manage.py createsuperuser
```

### Check Status
```bash
railway status
```

### View Environment Variables
```bash
railway variables
```

### Open Dashboard
```bash
railway open
```

## 🆘 COMMON ISSUES & FIXES

| Issue | Solution |
|-------|----------|
| Service fails to start | Check logs: `railway logs` |
| Can't connect to database | Verify `DB_HOST=postgres`, `DB_NAME=prompts` |
| Frontend can't reach backend | Set `BACKEND_URL=<backend-service-url>` |
| Migrations not running | Run manually: `railway exec backend python manage.py migrate` |
| Static files not loading | Run: `railway exec backend python manage.py collectstatic` |
